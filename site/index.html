API Explorer

Interactive query patterns for the infrastructure graph. Select parameters and click Run to see live results.
QUERY #ImpactQuery
Find all resources affected if target fails (blast radius analysis).
CUE: impact: #ImpactQuery & {Graph: infra, Target: "-"}
Target Node
Result: affected
QUERY #DependencyChain
Get the dependency path from target up to root(s).
CUE: chain: #DependencyChain & {Graph: infra, Target: "-"}
Target Node
Result: path
QUERY #PathFinder
Find shortest path between any two nodes in the graph.
CUE: path: #PathFinder & {Graph: infra, From: "-", To: "-"}
From Node
To Node
Result: path
QUERY #ImmediateDependents
Direct dependents only (not transitive). Nodes that directly depend on target.
CUE: deps: #ImmediateDependents & {Graph: infra, Target: "-"}
Target Node
Result: dependents
GET #CriticalityRank
Rank resources by dependent count (most critical first).
CUE: crit: #CriticalityRank & {Graph: infra}
Result: ranked
GET #GroupByType
Group resources by their @type array.
CUE: byType: #GroupByType & {Graph: infra}
Result: groups
GET #InfraGraph.topology
Layer ordering by dependency depth. Layer 0 = roots.
CUE: infra.topology
Result: topology
GET #ValidateGraph
Check for missing deps, self-refs, cycles, empty types.
CUE: validate: #ValidateGraph & {Input: resources}
Result

Dependency Layers (Timeline)

Using with CMDB/TopDesk data

For raw CMDB exports, use the quicue-adapters pipeline:

# 1. Place your CMDB JSON in adapters/topdesk/
# 2. Export as JSON-LD:
cue export ./adapters/topdesk/ -e jsonld --out json

# 3. Paste the output here

The adapter normalizes:

  • OS versions (103+ variations → canonical names)
  • Server types (VM, Physical, Container, etc.)
  • Network zones, SLA tiers, asset status
  • Infers @type from software/function fields
